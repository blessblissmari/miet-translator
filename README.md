# MIET Translator

Браузерный переводчик академических документов и презентаций (PDF, DOCX, PPTX, картинки) с английского на русский в стилистике МИЭТ. Развёрнут на GitHub Pages: <https://blessblissmari.github.io/miet-translator/>.

## Архитектура (Parse → Translate → Build)

Все три этапа крутятся прямо в браузере. Никакого бэкенда нет — данные уходят только в OpenRouter.

1. **Parse**. На входе — байты файла. На выходе — Markdown на исходном языке.
   - PDF с текстовым слоем: `pdfjs` достаёт текст и встроенные картинки (без LLM).
   - PDF, помеченный пользователем как «рукопись/скан», или PDF без текстового слоя: каждая страница рендерится в PNG и прогоняется через **vision-LLM как парсер** (`src/lib/visionParse.ts`). Промпт явно требует только распознать содержимое, без перевода. Математика — в `$...$` / `$$...$$`.
   - DOCX: `mammoth.js` (текст), картинки и формулы пока на минимуме.
   - PPTX: per-slide извлечение, JSON-промпт собирает `SlidePlan`.
2. **Translate** (`src/lib/translate.ts`). Markdown с исходного языка → Markdown на академическом русском, **отдельный** текстовый вызов LLM. Формулы и Markdown сохраняются дословно — модель видит готовый LaTeX и не пытается его «улучшить».
3. **Build**. Markdown → DOCX (`src/lib/docxBuild.ts`) или PPTX (`src/lib/pptxBuild.ts`). LaTeX-формулы конвертируются в OMML через `temml`.

Раньше парсинг и перевод делались одним вызовом — это уродовало формулы. Теперь две стадии независимы: одна модель занимается только OCR, вторая — только переводом.

### Пер-файловый флажок «Рукопись / скан»

В очереди слева у каждого файла есть чекбокс. Когда он включён, **все** страницы документа уходят через vision-LLM парсер (а не через `pdfjs`), даже если у PDF есть текстовый слой. Требует vision-капабельную модель (`Gemma 4 26B`, `Gemma 3 27B/12B`, `Nemotron VL`).

## Локальная разработка

```bash
npm install
cp .env.example .env.local   # пропиши свой OpenRouter key
npm run dev
```

`.env.local` гитом игнорируется. Можно вообще не создавать — UI спросит ключ и положит в `localStorage`.

## Деплой

Push в `main` → GitHub Actions собирает и публикует на GitHub Pages. Чтобы ключ OpenRouter попал в билд (и пользователи не вводили его сами), добавь в репозиторий секрет:

- Repo → Settings → Secrets and variables → Actions → New repository secret
- Name: `OPENROUTER_API_KEY`
- Value: твой ключ с <https://openrouter.ai/keys>

После этого следующий пуш в `main` запечёт ключ в бандл. Поле ввода ключа в UI остаётся как override.
